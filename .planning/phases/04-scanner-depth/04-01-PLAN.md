# Plan 04-01: Core Type Updates and OSV.dev Integration

## Objective
Establish foundational types for new findings and integrate the OSV.dev scanner logic into `package-scanner.ts`.

## Requirements
- SCD-02: Integrate with OSV.dev API to query for npm package CVEs and surface CRITICAL/HIGH findings and its tests.

## Tasks

### 1. Update `src/types/scan-result.ts` with new finding IDs and severities.
**<read_first>**
- `src/types/scan-result.ts`
**</read_first>**
**<action>**
Add `KnownVulnerabilityCritical` and `KnownVulnerabilityHigh` to the `FindingId` type definition and add them to the `FINDING_IDS` array if it exists.
**</action>**
**<acceptance_criteria>**
- `src/types/scan-result.ts` contains `type FindingId = ... | 'known-vulnerability-critical' | 'known-vulnerability-high';`
- If `FINDING_IDS` array exists, it contains `'known-vulnerability-critical'` and `'known-vulnerability-high'`.
**</acceptance_criteria>**

### 2. Implement OSV.dev API integration in `src/scanners/package-scanner.ts`.
**<read_first>**
- `src/scanners/package-scanner.ts`
- `src/types/scan-result.ts`
- `src/scanners/registry-scanner.ts` (for example of network call)
**</read_first>**
**<action>**
Modify the `scanPackage` function to:
- Make a POST request to `https://api.osv.dev/v1/query` with `{"package":{"name":"<pkg>","ecosystem":"npm"}}`.
- Implement a 5-second timeout using `AbortController`.
- Handle network errors and timeouts silently (NEVER crash, log a warning if verbose).
- Parse the response to identify vulnerabilities.
- For each vulnerability, if CVSS >= 9.0, add a finding with `id: 'known-vulnerability-critical', severity: 'CRITICAL'`.
- If CVSS >= 7.0 (and < 9.0), add a finding with `id: 'known-vulnerability-high', severity: 'HIGH'`.
**</action>**
**<acceptance_criteria>**
- `src/scanners/package-scanner.ts` imports `fetch` and `AbortController`.
- `scanPackage` function makes a POST request to `https://api.osv.dev/v1/query`.
- `scanPackage` handles a 5-second timeout for the fetch request.
- `scanPackage` silently handles fetch errors (e.g., network issues, timeouts).
- `scanPackage` correctly identifies and generates findings (`known-vulnerability-critical`, `known-vulnerability-high`) based on CVSS scores.
**</acceptance_criteria>**
