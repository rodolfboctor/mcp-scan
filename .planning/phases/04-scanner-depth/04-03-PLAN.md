# Plan 04-03: .env Leak Scanner Implementation

## Objective
Implement the .env leak scanner and create its corresponding tests.

## Requirements
- SCD-03: Implement a scanner to flag `.env` files containing secrets in server working directories and its tests.

## Tasks

### 2. Create `src/scanners/env-leak-scanner.ts` with initial structure and detection logic.
**<read_first>**
- `src/scanners/env-leak-scanner.ts` (will be new)
- `src/scanners/secret-scanner.ts` (for example scanner structure)
- `src/types/scan-result.ts`
**</read_first>**
**<action>**
Create `src/scanners/env-leak-scanner.ts` with an exported function (e.g., `scanEnvLeak`) that:
- Takes a `ResolvedServer` as input.
- Initializes an empty array of `Finding` objects.
- Determines parent directories of `server.configPath`.
- Checks for `.env` files in parent directories.
- Reads `.env` files and detects keys matching `*_KEY|*_SECRET|*_TOKEN|*_PASSWORD|API_*|AUTH_*` with values > 20 chars.
- Skips values containing `YOUR_`, `REPLACE`, `EXAMPLE`, `PLACEHOLDER`.
- Returns an array of `Finding` objects with `id: 'env-secret-exposed'` and `severity: 'HIGH'`.
**</action>**
**<acceptance_criteria>**
- File `src/scanners/env-leak-scanner.ts` exists.
- `src/scanners/env-leak-scanner.ts` exports `scanEnvLeak` function.
- `scanEnvLeak` function correctly:
  - Traverses parent directories to find `.env` files.
  - Reads content of `.env` files.
  - Identifies secret keys (KEY NAME ONLY) based on patterns.
  - Filters out placeholder values.
  - Returns `EnvSecretExposed` findings with `HIGH` severity.
- The scanner implementation is under 200 lines.
**</acceptance_criteria>**
