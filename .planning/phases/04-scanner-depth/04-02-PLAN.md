Implement the prompt injection scanner and create its corresponding tests.

## Requirements
- SCD-01: Implement a prompt injection scanner to detect patterns in tool descriptions and its tests.

### 2. Create `src/scanners/prompt-injection-scanner.ts` with initial structure and detection logic.
**<read_first>**
- `src/scanners/prompt-injection-scanner.ts` (will be new)
- `src/scanners/secret-scanner.ts` (for example scanner structure)
- `src/types/scan-result.ts`
**</read_first>**
**<action>**
Create `src/scanners/prompt-injection-scanner.ts` with an exported function (e.g., `scanPromptInjection`) that:
- Takes a `ResolvedServer` as input.
- Initializes an empty array of `Finding` objects.
- Contains the defined patterns for:
  - Strings: "ignore previous instructions", "ignore all prior", "you are now", "disregard", "forget your instructions", "override your"
  - Unicode: U+200B, U+FEFF, U+202E, U+00AD
  - Base64 > 50 chars: `/[A-Za-z0-9+/]{50,}={0,2}/`
  - Tool names: `bash`, `python`, `eval`, `exec`, `shell`, `terminal`, `run`, `system`
  - `additionalProperties: true` at schema root (this will require parsing the `server.schema` or `server.args` to look for this specific string).
- Returns an array of `Finding` objects with appropriate `id` and `severity` (HIGH, MEDIUM, LOW) as per `04-CONTEXT.md`.
**</action>**
**<acceptance_criteria>**
- File `src/scanners/prompt-injection-scanner.ts` exists.
- `src/scanners/prompt-injection-scanner.ts` exports `scanPromptInjection` function.
- `scanPromptInjection` function correctly identifies:
  - `prompt-injection-pattern` (HIGH) for string patterns.
  - `unicode-injection` (HIGH) for Unicode patterns.
  - `tool-name-shadow` (MEDIUM) for tool name patterns.
  - `schema-bypass-risk` (LOW) for `additionalProperties: true` at schema root.
- The scanner implementation is under 200 lines.
**</acceptance_criteria>**
